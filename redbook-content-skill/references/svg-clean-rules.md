# SVG 清洁规则：小红书图文的「少即是多」

> 从实际踩坑中提炼的经验。小红书图文的核心是**内容干净、信息聚焦**，SVG 中任何多余元素都会降低专业感和可读性。

---

## 一、SVG 中不放的内容（红线清单）

### 1. 不放 # 标签
- **原因**：标签在图中会占用宝贵版面，且与 caption 中的标签重复。小红书用户看图获取信息，标签是给算法看的，不该出现在图里。
- **做法**：所有 `#话题` 标签只放在 `caption.md` 的文案末尾。

### 2. 不放引导收藏/关注的文案
- **原因**：图中放「关注我」「收藏备用」「点赞转发」等引导语，会被平台判定为营销内容，降低推荐权重。而且这类文案对用户没有实际信息价值。
- **做法**：引导互动的 CTA 只放在 `caption.md` 文案结尾，用 emoji 列表自然引导（如 `👉 关注我，每周更新`）。

### 3. 不放账号名/品牌名
- **原因**：图中反复出现账号名（如「AI干货铺」「@xxx」）会显得像硬广，降低内容可信度。小红书用户更信任「内容本身好」而非「品牌反复刷脸」。
- **做法**：所有图中不出现任何账号名、品牌名、@信息。账号信息只在发布时由平台自动关联。

### 4. 不放页码
- **原因**：页码是 PPT 思维，小红书图文是九宫格流式浏览，用户通过滑动切换，页码是多余的视觉噪音。
- **做法**：所有 SVG 中不出现「1/9」「第1页」等页码标记。

---

## 二、正文颜色必须够深

### 规则
- 正文 fill 必须用深色（如 `#2E2A24`、`#2B2B2B`、`#1F2A44`），确保手机屏幕可读。
- 禁止用浅灰（如 `#999`、`#AAA`、`#8C7B6B`）做正文颜色——在手机小屏幕上几乎看不清。
- 浅色仅限装饰性文字（标注、页脚等非核心信息）。

### 为什么
- 小红书用户在手机上浏览，屏幕小、环境光复杂，浅色文字在强光下几乎不可见。
- 对比度 ≥ 4.5:1（WCAG AA）是硬指标，但实际体验中，正文用深色（对比度 ≥ 7:1）才够舒适。

---

## 三、XML 实体转义（致命错误）

### 规则
- SVG 是 XML 格式，以下字符必须转义，否则浏览器/渲染器会报错并无法显示：
  - `&` → `&amp;`（**最常见坑**，如"对话 & 研究"必须写成"对话 &amp; 研究"）
  - `<` → `&lt;`
  - `>` → `&gt;`
  - `"` → `&quot;`（在属性值内时）
- **只在 `<text>` 元素的文本内容中需要转义**，属性值中的引号用 `&quot;` 即可。

### 为什么
- XML 解析器遇到裸 `&` 会尝试将其解析为实体引用（如 `&amp;`），若后面跟的不是合法实体名，直接报错 `xmlParseEntityRef: no name`，整个 SVG 无法渲染。
- 这是 SVG 最常见的致命错误，生成时务必逐个检查 `<text>` 中的 `&`。

### 检查清单
- [ ] 所有 `<text>` 内容中的 `&` 已替换为 `&amp;`
- [ ] 所有 `<text>` 内容中的 `<` 已替换为 `&lt;`
- [ ] 所有 `<text>` 内容中的 `>` 已替换为 `&gt;`

---

## 四、禁止文本重叠（编辑残留）

### 规则
- 同一坐标 (x, y) 上**不允许存在两个 `<text>` 元素**，否则文字完全重叠、不可读。
- 这通常发生在**修改过程中**：新版本写入后忘记删除旧版本，导致同一位置留有两行文字。
- 生成/修改 SVG 后，必须逐行扫描 y 坐标，确认无重复。

### 常见模式（危险信号）
```xml
<!-- ❌ 两行文字都在 y=630，完全重叠 -->
<text x="140" y="630" ...>关键指标：任务完成率</text>
<text x="140" y="630" ...>难点：操作路径多样</text>
```

### 检查清单
- [ ] 所有 `<text>` 的 y 坐标无完全重复（同一 y 出现两次即报警）
- [ ] 编辑修改后，旧版本文字已完全删除（非注释，是删除）

---

## 五、文字不溢出容器边界

### 规则
- `<text>` 元素的 y 坐标必须在所属容器 `<rect>` 的 y ~ y+height 范围内。
- 典型错误：容器高度不够，最底部的文字行超出矩形边界。
- 修改容器内容后，必须**同步调整容器高度**。

### 检查方法
```
容器: rect y=300, height=400 → 底部 y=700
文字: text y=730 → ❌ 溢出 30px
修复: 调整 height=450 → 底部 y=750，文字 y=730 在容器内 ✅
```

### 检查清单
- [ ] 容器内所有 `<text>` 的 y 坐标 ≤ rect.y + rect.height - 20（留 20px 底部 padding）
- [ ] 修改内容后，容器高度已同步更新

---

## 六、经验总结：小红书图文的「减法原则」

| 做法 | 原因 |
|------|------|
| 图中只放**内容信息** | 用户看图是为了获取干货，不是看广告 |
| 标签/CTA/账号名放文案区 | 图文分离，各司其职：图=信息，文=引导 |
| 正文用深色 | 手机可读性是第一优先级 |
| 不放页码 | 九宫格流式浏览，页码是噪音 |
| 少即是多 | 每多一个非内容元素，专业感降一分 |
