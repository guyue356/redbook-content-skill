# 小红书图文 QA 门禁

> 对应 ppt-master 的 `svg_quality_checker.py` 质量门禁，但改为小红书专属规则（人工对照，非脚本强制）。
> 在 `svg_to_png.py` 导出后、交付前逐图执行。**error 必改，warning 能改就改。**

---

## error 级（必须修复）

| 检查项 | 规则 | 说明 |
|---|---|---|
| 安全区 | 文字/关键元素不入顶部 140px、底部 180px 区 | 防被 APP UI 遮挡 |
| 不溢出 | 所有元素落在 0–1242 × 0–1660 内 | 无裁切、无出血 |
| 字号恒定 | 同角色（封面大字/小标题/正文/标注/页脚）跨图 px 一致 | 防「AI 感」漂移 |
| 配色越界 | fill/stroke 仅用 `redbook_spec.md` 配色 + 中性灰白 | 禁止临场新色 |
| 封面钩子 | 封面含主标题钩子 + 利益点/副标题 | 缺任一即不合格；不含账号名 |
| 移动可读性 | 正文 ≥34px、标注 ≥24px | 低于手机糊 |
| 九宫格完整 | 至少 3 图；末图为结尾CTA | 少于 3 图或结尾缺失判不合格 |
| 标签数 | caption 含 3–8 个 #话题 | 无标签/超量降权 |
| SVG 无标签 | SVG 图中不含 `#话题` 标签文字 | 标签只放 caption.md，图中出现会遮挡内容且降低可读性 |
| SVG 无引流文案 | SVG 图中不含「关注」「收藏」「点赞」「转发」等引导互动文案 | 引导互动只放 caption.md 文案，图中放会被平台判定营销降权 |
| SVG 无品牌水印 | SVG 图中不含「AI干货铺」等任何账号/品牌推广字样 | 所有图中不出现账号名、品牌名、@信息 |
| XML 实体转义 | `<text>` 内容中 `&` 必须为 `&amp;`，`<` 为 `&lt;`，`>` 为 `&gt;` | 裸 `&` 导致 `xmlParseEntityRef: no name`，SVG 完全无法渲染 |
| 无文本重叠 | 同一 (x, y) 坐标不出现两个 `<text>` 元素 | 编辑残留导致两行文字完全重叠，不可读 |
| 文字不溢出容器 | `<text>` 的 y ≤ 所属 `<rect>` 的 y + height - 20 | 容器高度不足或修改内容后未同步调整高度，最底部文字溢出边界 |

---

## warning 级（建议修复）

| 检查项 | 规则 |
|---|---|
| 对比度 | 文字/背景 ≥ 4.5:1；弱对比加描边或底块 |
| emoji 一致 | 同语义 emoji 跨图统一，不混搭 |
| 节奏单调 | 连续 ≥3 张 dense 无 breathing 金句页 → 插一张 |
| 字体缺失风险 | 字体栈未以预装字体收尾 |
| 标题党 | 封面/标题承诺与内容不符 |

---

## 自检清单（交付前逐项勾）

- [ ] 每张图重读过 `redbook_spec.md`
- [ ] 封面含主标题钩子 + 利益点，一眼懂（不含账号名）
- [ ] 安全区无违规
- [ ] 同角色字号跨图一致
- [ ] 配色仅用锁定值
- [ ] 正文 ≥34px、无溢出
- [ ] **`&` 已转义为 `&amp;`（逐个检查所有 `<text>` 元素）**
- [ ] **无同 y 坐标重叠文本（编辑后逐行扫描）**
- [ ] **所有文字在容器边界内（修改内容后同步调容器高度）**
- [ ] 末图 CTA 明确
- [ ] caption 含 3–8 标签，大词+长尾结构
- [ ] PNG 导出清晰（建议 2x = 2484×3320）

---

## 与 ppt-master 质检的差异

- ppt-master 用脚本强制（viewBox/字体/漂移自动报错）；本 skill 因产物是 PNG 且需人工审美判断，改为**对照清单**人工门禁。
- 新增小红书特有项：**安全区、移动可读性、封面钩子、九宫格完整、标签数**。
